### Adding Fractions — full detail fetched
URL: https://www.openmiddle.com/adding-fractions/
Grade 5, Standard 5.NF.1. "Using the digits 1 to 9, at most one time each, place a digit in each box to make the smallest (or largest) possible sum." Boxes form: □/□ + □/□. Hint given: to maximize, maximize numerators/minimize denominators (improper fractions allowed); to approach a target like 1, use near-equal fraction pairs. Answers: largest = 9/1 + 8/2, smallest ≈ 2/9 + 1/8 (~0.347). Source credit: Robert Kaplinsky. DOK 3 (Strategic Thinking).

## Why this source matters for the plan
Every Open Middle problem uses the SAME underlying skill as a standard textbook question (add fractions, find area/perimeter) but reframes it as "find the digit arrangement that satisfies a constraint" — genuine reasoning/optimization instead of one-shot computation. This is a direct, legitimate model for recalibrating `m_words`/`m_bar` generators to be harder without changing the underlying grade-level skill being tested — exactly the "escalate real rigor, not just bigger numbers" gap flagged by the user.
