## Changes made in `growth-map.html`

1. **`m_words`**: added `else if(L.algebra>=1){...}` after the existing `algebra>=2` branch (order matters — `>=2` is checked first so rungs 10-12 keep getting the mean/median problem unchanged; only rung 9's exact `algebra:1` falls through to the new branch). New branch: a "think of a number" word problem — pick a name, a multiplier `m` (2-6), an addend `c` (2-15), and a hidden number `x` (3-15); state the result `m·x+c` and ask for `x`. Always exact (all integers).
2. **`m_mistake`**: added `algebraBank` (two items — the "▢+6=15, so ▢=15+6=21" undo-addition error, and the "3x=21, so x=21+3=24" undo-multiplication error, both classic inverse-operation misconceptions), merged in via `if(L.algebra>=1)bank=bank.concat(algebraBank);`. This reaches rungs 9, 10, 11, and 12 (all four set `algebra>=1`) — a deliberately broad reach matching how broadly the flag itself is set, same additive pattern already used for `fracEasy`/`decBank`/`ratioBank`/`pctBank`.
3. **`m_words`' steps block** (`L.s>=3`, non-decimal branch): changed `per`/`m`/`n` ranges from fixed `R(3,7)`/`R(4,15)` to `L.longmul?R(6,15):R(3,7)` / `L.longmul?R(10,40):R(4,15)` — scales up for every `longmul` rung (6 through 12), fixing the flat-range gap described above. Rungs without `longmul` (1-5, and Mirayah entirely, which uses separate functions) are byte-for-byte unaffected — verified below.
4. **`m_words`' change block** (always-present, non-decimal branch): changed the `L.mul2` ternary to `(L.mul2||L.longmul)`, same rationale — scales the "pens cost ₹X, pay with a note" problem up for `longmul` rungs too, not just `mul2` rungs.

No existing function signatures or return shapes changed; `ws_maths_sarah`'s call signature is untouched. Knob object itself (`LEVELS.sarah.maths[8]`) is **unchanged** — the audit confirmed it was already correctly configured; every fix this rung needed was in generator reach, not knob values.
